Overview

In this work, we investigate the sample mean and sample variance from the exponential distribution; and we compare their expectations from the simultation with the population mean and population variance, respectively. To further check the variability of the sample mean, we look into the variance of the sample mean and compare with the logical estimate from the Central Limit Theorem (CLT).

Simulations

We choose the exponential distribution with the probability density function \(p(x) = \lambda \exp^{-\lambda x}\). In all the simulations, \(\lambda = 0.2\).

In total, we run num_simulations simulations, in each of which there are num_samples samples.

set.seed(0)
run_simulation <- function(num_simulations, num_samples, eval_option, lambda=0.2) {
    sample_means = NULL
    sample_variances = NULL
    for (sim_idx in 1:num_simulations) {
        sim_data = rexp(num_samples, rate = lambda)
        sample_means = c(sample_means, mean(sim_data))
        sample_variances = c(sample_variances, var(sim_data))
    }
    sample_means <- data.frame(mean=sample_means) 
    sample_vars <- data.frame(var=sample_variances)
    results <- NULL
    if(eval_option=="mean")
        results <- sample_means
    if(eval_option=="var")
        results <- sample_vars
    return(results)
}

Comparison between Sample Mean and Population Mean

The population mean \(\mu\) is \(\frac{1}{\lambda}\), which can be calculated analytically. According to the CLT, the expectation of the sample mean is equal to \(\mu\); the variance of the sample mean is \(\frac{\sigma^2}{n}\), where \(\sigma^2\) is the population variance and \(n\) is the number of samples.

Here we show the distribution of the sample mean. Note the red line denotes the population mean.

The comparison is further shown in the following table.

n expectated sample mean population mean
10 5.0304449 5
20 4.949264 5
40 5.0198671 5

The results suggest that the expectation of the sample mean is approaching to population mean when the number of samples increases.

Comparison between Sample Variance and Population variance

The popluation variance \(\sigma^2\) is \(\frac{1}{\lambda^2}\).

We plot the distribution of the sample variance in black. For comparison, the population mean is plotted in red line.

The comparison is further shown in the following table.

n expectated sample variance population variance
10 25.0874853 25
20 24.6951726 25
40 24.5016185 25

The results suggest that the expectation of the sample variance is approaching to population variance when the number of samples increases. Moreover, the distributions are increasingly concerntrated. We also note that the distributions are not normal distributions and are skewed.

The Variance of Sample Mean

According to CLT, the variance of the sample mean is \(\frac{\sigma^2}{n}\). We plot the distribution of the logical estimates of the variances, i.e. \(\frac{sample variance}{n}\).

The comparison is further shown in the following table.

n average of the logical estimates variance of sample mean
10 2.5087485 2.50
20 1.2347586 1.25
50 0.6125405 0.625

The results suggest that logical estimate of the sample mean is a reasonable estimate. The sample mean become less variable, when the number of samples increases, because and the difference between this estimate and the estimand becomes increasingly smaller. As the distribution of the sample variance, these distributions are not normal distribution and are skewed.

Conclusion

  • We show that the distribution of sample mean are approaching to a normal distribution when the number of samples increases. Meanwhile,
    • the expectation of sample mean is approaching to population mean;
    • the sample mean become less variable.
  • The distribution of sample variance is not normal distributed and skewed. But their expectation is approaching to the population variance.
  • These results are consistent with the CLT.

library(ggplot2)
# Multiple plot function
#
# ggplot objects can be passed in ..., or to plotlist (as a list of ggplot objects)
# - cols:   Number of columns in layout
# - layout: A matrix specifying the layout. If present, 'cols' is ignored.
#
# If the layout is something like matrix(c(1,2,3,3), nrow=2, byrow=TRUE),
# then plot 1 will go in the upper left, 2 will go in the upper right, and
# 3 will go all the way across the bottom.
#
# Reference: http://www.cookbook-r.com/Graphs/Multiple_graphs_on_one_page_(ggplot2)/
multiplot <- function(..., plotlist=NULL, file, cols=1, layout=NULL) {
    require(grid)
    
    # Make a list from the ... arguments and plotlist
    plots <- c(list(...), plotlist)
    
    numPlots = length(plots)
    
    # If layout is NULL, then use 'cols' to determine layout
    if (is.null(layout)) {
        # Make the panel
        # ncol: Number of columns of plots
        # nrow: Number of rows needed, calculated from # of cols
        layout <- matrix(seq(1, cols * ceiling(numPlots/cols)),
                         ncol = cols, nrow = ceiling(numPlots/cols))
    }
    
    if (numPlots==1) {
        print(plots[[1]])
        
    } else {
        # Set up the page
        grid.newpage()
        pushViewport(viewport(layout = grid.layout(nrow(layout), ncol(layout))))
        
        # Make each plot, in the correct location
        for (i in 1:numPlots) {
            # Get the i,j matrix positions of the regions that contain this subplot
            matchidx <- as.data.frame(which(layout == i, arr.ind = TRUE))
            
            print(plots[[i]], vp = viewport(layout.pos.row = matchidx$row,
                                            layout.pos.col = matchidx$col))
        }
    }
}
